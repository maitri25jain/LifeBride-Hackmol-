# 🫀 LifeBridge

**Decentralized, Scoring-Driven Organ, Blood & Plasma Donation Platform for India**

LifeBridge replaces India's slow, manual, and often biased organ allocation system with a high-velocity ecosystem powered by Rule-based scoring matching, blockchain-verified consent, and emergency blood donor broadcasts.

---

## The Problem

India's National Organ & Tissue Transplant Organization (NOTTO) still relies on basic SQL databases and manual phone calls. This leads to:

- **Viability Loss** — Organs expire while coordinators dial hospitals. A heart has only 4 hours of cold ischemia time.
- **Consent Disputes** — Paper pledges are lost, forged, or disputed by grieving families in the ICU.
- **Systemic Bias** — Documented socioeconomic inequities in who gets matched first.
- **Donor Attrition** — No engagement ecosystem for blood donors beyond the first donation.

Current manual matching takes **6–8 hours**. LifeBridge targets to match **under 30 minutes**.

## Key Features

### 1. Immutable Consent — Soulbound NFT (ERC-5192)
- Donor pledges minted as **non-transferable** ERC-721 tokens on Polygon
- Linked to SHA-256 hashed Aadhaar identity — no raw PII on-chain
- `isPledgeActive()` lets hospitals verify consent in seconds
- Nobody can forge, delete, or transfer a pledge

### 2. Rule-based S Matching Engine
- **Organ matching**: Scores recipients on HLA compatibility (40%), urgency (30%), waitlist equity (20%), ABO rules
- **Viability clock**: Cold ischemia countdown with haversine transport time estimation
- **Blood matching**: ABO compatibility, geo-radius filtering, 56-day cooldown check
- Decoupled microservice — swappable for trained ML model without backend changes

### 3. Emergency Blood Broadcast
- Coordinators broadcast blood emergencies with type, urgency, and radius
- Citizens with matching blood type within radius see alerts on dashboard
- One-tap response — coordinator sees responder list instantly

### 4. Role-Based Hospital Command Centre
- **Coordinators**: Register donors/recipients, run AI matching, broadcast alerts
- **Admins**: Coordinator privileges + account management
- Every action scoped to hospital via JWT

## Quick Start

### Prerequisites
- Node.js 18+ · Python 3.11+ · MongoDB Atlas free cluster

### 1. Backend (Terminal 1)
```bash
cd backend
touch app/__init__.py app/api/__init__.py app/core/__init__.py app/schemas/__init__.py
pip install fastapi uvicorn pymongo python-dotenv "python-jose[cryptography]" "passlib[bcrypt]" pydantic httpx web3
# Create .env with MONGO_URL, SECRET_KEY, MATCHING_ENGINE_URL
python -m app.core.seed
uvicorn main:app --reload --port 8000
```

### 2. AI Engine (Terminal 2)
```bash
cd matching-engine
pip install fastapi uvicorn pydantic
uvicorn main:app --reload --port 8001
```

### 3. Frontend (Terminal 3)
```bash
cd frontend
npm install
npm run dev
```

### 4. Blockchain — Optional (Terminal 4)
```bash
cd smart-contracts
npm install
npx hardhat node
# New terminal:
npx hardhat ignition deploy ./ignition/modules/deploy.ts --network localhost
```

Open **http://localhost:5173**

## Security

- Aadhaar: SHA-256 hashed before storage
- Passwords: bcrypt with automatic salting
- JWT: 24-hour expiry, role-based guards
- Hospital scoping: Data filtered by hospital_id from JWT
- Blockchain: Soulbound tokens — no transfer, no forgery
- Audit trail: Every action logged with user + timestamp
